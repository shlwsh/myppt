## 二、双算法清单与数据集映射

### 2.1 算法 A：Edge-IQA（边缘图像质量评估）

| 项 | 内容 |
|----|------|
| 问题 | 低质量舌象/面象上传云端 → LLM 闭环延迟高、GIGO |
| 方法 | 轻量级 **清晰度（Laplacian 方差）+ 曝光（直方图熵/过曝比例）** 融合得分 $Q_{img}\in[0,1]$ |
| 实现 | 边缘网关 Python/TS 模块；单帧推理 &lt; 30ms（CPU 目标） |
| 训练数据 | TCM-Tongue `train/val`；可选在模糊/曝光增强上自监督标定阈值 |
| 输出 | `Q_img`、失败原因标签 `{blur, dark, overexposed}` |

### 2.2 算法 B：Confidence-Aware Routing（LangGraph 自适应路由）

| 项 | 内容 |
|----|------|
| 问题 | 固定 FSM 无法按质量动态选择「边缘重拍 vs 上传云端」 |
| 方法 | 条件边：`route(state) → resample_edge \| upload_cloud \| fail_safe`，依据 $Q_{img}$ 与阈值 $\tau$ |
| 实现 | LangGraph `StateGraph`；节点：`capture` → `edge_iqa` → `route` → `resample` / `cloud_vision` |
| 对比基线 | B0 全云端 IQA；B1 固定阈值无路由；B2 本文完整方法 |
| 输出 | 路由决策日志、状态转移序列（用于可复现） |

### 2.3 数据集

| 数据集 | 路径 | 论文用途 |
|--------|------|----------|
| **TCM-Tongue**（主） | `doctor/datasets/vision/tcm-tongue/` | Edge-IQA 标定/验证；仿真视觉靶标 |
| **TCM-FD**（辅） | `doctor/datasets/vision/tcm-fd/` | 跨数据集泛化（曝光/肤质差异） |
| 仿真 | `doctor/datasets/sim/ros2-gazebo/` | RTT、重拍次数、网络抖动注入 |

## 三、量化指标与实验假设（投稿前必须填满）

| 指标 ID | 定义 | 目标方向 | 主图 |
|---------|------|----------|------|
| M1 | 闭环 RTT（采集→IQA→路由→动作）p50/p95 | ↓ ≥30% vs B0 | Fig.5 RTT CDF |
| M2 | 有效采集率（$Q_{img}\geq\tau$ 且任务完成） | ↑ ≥15% vs B0 | Fig.6 柱状图 |
| M3 | 云端无效上传占比 | ↓ ≥40% vs B0 | 表格 |
| M4 | $\tau$ 消融：有效率 vs 边缘负载 | U 型最优 $\tau^*$ | Fig.7 |
| M5 | TCM-FD 上 $Q_{img}$ 与人工一致率（抽样 200 张） | Spearman ≥0.75 | 补充材料 |

**统计**：每配置 ≥3 随机种子 × 每集 ≥500 帧（或 50 次仿真 trial）；报告 mean±std。
