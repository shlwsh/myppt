# 开题报告文献 PDF 批量下载：流程与要点

本文档基于 `博士论文开题报告20260524-007/008` 的文献整理与下载实践整理，供后续开题报告版本复用。

---

## 1. 目标与产出

| 产出 | 说明 |
|------|------|
| `pdf00N/` 目录 | 与报告版本对应的本地 PDF 库（如 `pdf008`） |
| `download_manifest.json` | 每条链接的下载结果（成功/失败/字节数/实际 PDF URL） |
| `README.md`（可选） | 人工可读的汇总说明 |
| 报告第七章链接 | 可下载的真实 DOI / arXiv / PMC / IEEE 等地址 |

---

## 2. 总体流程（推荐顺序）

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│ 1. 审查文献链接  │ ──► │ 2. 修正占位/无效URL │ ──► │ 3. 统一引用格式   │
└─────────────────┘     └──────────────────┘     └────────┬────────┘
                                                          │
┌─────────────────┐     ┌──────────────────┐              ▼
│ 6. 更新报告链接  │ ◄── │ 5. 补全失败条目    │ ◄── ┌─────────────────┐
└─────────────────┘     └──────────────────┘     │ 4. 批量下载 PDF   │
                                                  └─────────────────┘
```

### 步骤 1：审查文献链接

从 Markdown **第七章「文献参考」** 提取全部 URL，检查：

- 是否含 `google.com/search?q=`（**占位链接，必改**）
- arXiv 是否为 `abs/数字` 形式（如 `2308.08155`），而非描述性 slug（如 `med-dialogue-2025`）
- IEEE 是否为 `document/数字`，而非 `document/ros2-jazzy-perf-2024`
- 是否为博客/教程（Medium、TowardsAI、EmergentMind）——通常**无 PDF**

### 步骤 2：修正无效链接

**可靠来源优先级：**

1. **arXiv** — `https://arxiv.org/abs/{id}` → PDF：`https://arxiv.org/pdf/{id}.pdf`
2. **DOI** — `https://doi.org/10.xxxx/...`（配合 Unpaywall 查开放获取）
3. **PMC** — `https://pmc.ncbi.nlm.nih.gov/articles/PMC{id}/`（常需配合期刊全文页）
4. **Frontiers / BMC / MDPI** — 开放获取期刊，可用 `/pdf` 或 `content/pdf/{doi}.pdf`
5. **IEEE Xplore** — `document/{arnumber}` → `stampPDF/getPDF.jsp?arnumber={arnumber}`（部分需机构权限）

可参考同项目 `博士论文开题报告20260523-003.md` 中已验证的 DOI/arXiv 列表进行主题对齐替换。

### 步骤 3：统一引用格式（便于脚本解析与人工阅读）

推荐与 003/008 修订版一致：

```markdown
1. **论文标题 (年份)**
   [PDF 原文 (Local)](pdf008/02_abs_2601.21011.pdf) | [在线资源链接](https://arxiv.org/abs/2601.21011)
```

- 主链接使用 `[在线资源链接](真实URL)`
- 已有本地文件时增加 `[PDF 原文 (Local)](相对路径)`
- 避免仅写裸 URL 与 Google 搜索包装链接混用

### 步骤 4：批量下载

**环境要求：** Node.js ≥ 18（内置 `fetch`）

```bash
# 项目技能脚本（推荐）
node .agents/skills/thesis-reference-pdf-downloader/scripts/download_refs.mjs \
  --md doctor/博士论文开题报告20260524-008.md \
  --out doctor/pdf008

# 或 doctor 目录下的便捷封装（若存在）
node doctor/download_refs_008.mjs
```

脚本行为：

- 仅解析第七章及之后内容中的 URL
- 按域名生成候选 PDF URL 并依次尝试
- 校验文件头为 `%PDF`
- 请求间隔约 400ms，降低封禁风险
- 输出 `download_manifest.json`

### 步骤 5：处理失败条目

| 失败类型 | 典型原因 | 处理建议 |
|----------|----------|----------|
| HTTP 404 | 占位 URL、文档 ID 错误 | 查 DOI/arXiv 替换链接 |
| HTTP 403 | 出版社反爬、需订阅 | 机构 VPN；Unpaywall 查 OA；复制 `pdf/` 已有文件 |
| not PDF (text/html) | 落到摘要页、PMC 验证码页 | 换 Frontiers/Nature 全文 PDF 链 |
| fetch failed | 网络/防火墙 | 重试；换 DOI 镜像 |

**PMC 补充：** 可用 NCBI OA API 查是否有 OA 包（常为 FTP tar.gz，非直接 PDF）：

`https://www.ncbi.nlm.nih.gov/pmc/utils/oa/oa.fcgi?id=PMC{id}`

**Frontiers：** 将 `/full` 换为 `/pdf`，或使用 DOI 路径 `https://www.frontiersin.org/articles/{doi}/pdf`。

### 步骤 6：回写报告与 README

- 将成功下载的文件名写入 `[PDF 原文 (Local)](...)`
- 在 `pdf00N/README.md` 记录成功数、失败分类
- 勿使用 Sci-Hub 等未授权渠道

## 3. 各出版源 PDF 获取要点

| 来源 | 页面链接示例 | PDF 候选策略 |
|------|----------------|---------------|
| arXiv | `arxiv.org/abs/2403.01249` | `arxiv.org/pdf/2403.01249.pdf` |
| IEEE | `ieeexplore.ieee.org/document/11048919` | `stampPDF/getPDF.jsp?arnumber=11048919` |
| Springer OA | `link.springer.com/article/10.1186/...` | `content/pdf/10.1186/....pdf` |
| Frontiers | `.../articles/10.3389/.../full` | `.../articles/10.3389/.../pdf` |
| PMC | `pmc.ncbi.nlm.nih.gov/articles/PMC123.../` | 优先期刊官网 PDF；PMC `/pdf/` 常返回 HTML |
| preprints.org | `.../manuscript/202410.1204` | 尝试 `/v2/download`；403 时改 DOI 或 arXiv 替代 |
| DOI 聚合 | `doi.org/10.xxxx` | Unpaywall API（合法 OA 定位） |

## 5. 实践经验摘要（007 → 008）

| 现象 | 结论 |
|------|------|
| 007 版 50 个链接全部失败 | URL 为 AI 生成的描述性 slug，非真实文献 ID |
| 008 版部分真实链接 | arXiv、IEEE 数字 ID、Frontiers、Springer OA 可自动下载 |
| 008 版混有旧占位符 | 7.2 后半、7.4、7.5 需整段替换为 003 版已验证文献 |
| 6/50 → 修订后 | 修正链接 + 复用 `pdf/` 本地库，可覆盖绝大多数核心文献 |
| Windows 环境 | 优先 Node.js；PowerShell 注意 UTF-8 中文路径；避免 `&&` 用 `;` |

## 6. 合规与边界

- 仅使用出版商开放获取、arXiv、机构已订阅通道
- 不使用 Sci-Hub 等盗版站点
- 付费论文在报告中保留 DOI，本地 PDF 标注来源与权限限制
- 博客/技术文章可作为**参考链接**，不应标为「PDF 原文」
