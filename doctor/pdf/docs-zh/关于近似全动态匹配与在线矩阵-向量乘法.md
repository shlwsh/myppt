## 摘要

我们研究了全动态 $(1 - \epsilon)$-近似最大二分图匹配（Fully-Dynamic $(1 - \epsilon)$-Approximate Maximum Bipartite Matching）与对偶 $(1 + \epsilon)$-近似顶点覆盖（Approximate Vertex Cover）问题之间的联系，并结合了最近在若干细粒度硬度归约（Fine-Grained Hardness Reductions）中被广泛使用的在线矩阵-向量（Online Matrix-Vector, OMv）猜想。我们证明，当且仅当 OMv 猜想为假时，存在一种在线算法，可以在全动态更新下以摊销（amortized） $n^{1-c}\epsilon^{-C}$ 时间（对于常数 $c, C > 0$）维护 $(1 + \epsilon)$-近似顶点覆盖。类似地，我们证明，当且仅当对另一个我们称为动态近似 OMv 的、看似没有匹配结构的动态问题存在非平凡算法时，存在一种在线算法，可以在摊销 $n^{1-c}\epsilon^{-C}$ 时间内维护 $(1 - \epsilon)$-近似最大二分图匹配。这为反驳在近似全动态匹配和顶点覆盖问题中实现摊销次线性更新时间（Sublinear Update Times）提供了有力证据。

利用这些关联，我们在在线和离线设置下，获得了更快的高效近似全动态匹配算法：
* 我们给出一个随机算法（Randomized Algorithm），能够以极高概率在全动态图上以摊销 $O(\epsilon^{-O(1)} \frac{n}{2^{\Omega(\sqrt{\log n})}})$ 的更新时间维护 $(1 - \epsilon)$-近似二分图匹配和 $(1 + \epsilon)$-近似顶点覆盖。这改善了此前由 Assadi-Behnezhad-Khanna-Li [STOC 2023] 实现的 $O(n/(\log^* n)^{\Omega(1)})$ 以及由 Bhattacharya-Kiss-Saranurak [FOCS 2023] 在小 $\epsilon$ 下实现的 $O_\epsilon(n^{1-\Omega_\epsilon(1)})$ 的最快运行时间。我们的算法利用了 Larsen 和 Williams [SODA 2017] 提出的快速在线矩阵-向量乘法算法。
* 我们通过使用快速矩阵乘法（Fast Matrix Multiplication），给出了一个用于 $(1 - \epsilon)$-近似最大匹配的离线随机算法，其摊销运行时间为 $O(n^{0.58}\epsilon^{-O(1)})$，显著优于通过上述在线算法实现的运行时间。这与 OMv 的情况相呼应，其中离线算法恰好对应于快速矩阵乘法。我们还给出了一个离线算法，可以在摊销 $O(n^{0.723}\epsilon^{-O(1)})$ 时间内维护 $(1 + \epsilon)$-近似顶点覆盖。

## 1. 引言

令 $G = (V = L \cup R, E)$ 表示一个经受边缘插入和删除更新的二分图（Bipartite Graph）。最近，在 $G$ 中维护 $(1 - \epsilon)$-近似二分图匹配的问题受到了显著关注。对于这个动态问题，我们目前对最佳复杂度的理解还不够透彻。在更受限制的部分动态设置下，其中图仅经历边缘插入（增量式）或仅经历边缘删除（减量式），已知的算法具有 $\widetilde{O}(\epsilon^{-O(1)})$ 的摊销更新时间 [Gup14, JJST22, BKS23b]，甚至适用于加权泛化。此外，在诸如组合布尔矩阵乘法猜想或在线矩阵-向量（OMv）猜想等流行的硬度假设下，已知即使对于部分动态的无权图，精确版本也需要摊销 $\Omega(n^{1-o(1)})$ 的时间。

因此，最近的注意力转移到了在全动态（Fully Dynamic）设置中 $(1 - \epsilon)$-近似匹配的复杂度上，其中图可以同时进行边缘插入和删除。在这种设置下，一个“理想”的算法将以摊销 $O((\epsilon^{-1} \log n)^{O(1)})$ 的时间完成每次更新。此类算法的存在性此前已被作为一个公开问题提出 [ARW17]。然而，时至今日，甚至不知道对于与 $\epsilon$ 无关的某些绝对常数 $c < 1$，是否存在摊销时间为 $O_\epsilon(n^{1-c})$ 的算法。最著名的上限是 [GP13]，对于具有 $m$ 条边的图，更新时间为 $O(\epsilon^{-2} \sqrt{m})$。对于稠密图（这已被视为困难的情况），其运行时间为 $O(\epsilon^{-2} n)$。最近，利用 Szemeredi 正则引理，[ABKL23] 给出了一种在 $\epsilon = (\log^* n)^{-c}$ 下摊销更新时间为 $O(n/(\log^* n)^c)$（对于某些常数 $c > 0$）的算法。然而，由于 Behrend 构造 [Beh46]，基于正则性的方法不太可能节省超过 $2^{O(\sqrt{\log n})}$ 的因子。据作者所知，这些仍是随着 $\epsilon \to 0$ 维护匹配的最佳已知上限。最近，[BKS23a] 给出了一种可以在摊销 $O_\epsilon(m^{1/2-\Omega_\epsilon(1)}) = O_\epsilon(n^{1-\Omega_\epsilon(1)})$ 时间内仅维护匹配大小（Size Only）的算法。

鉴于缺乏上限，探讨是否能够证明下限是很有意义的。由于证明无条件硬度下限非常困难，目前流行的一种趋势是通过规约到某些著名硬度假设来证明下限。其中一个突出的假设是 [HKNS15] 提出的在线矩阵-向量（OMv）猜想。该猜想考虑以下问题：给出一个算法和一个布尔矩阵 $M \in \{0, 1\}^{n \times n}$，该矩阵在多项式时间内进行预处理。然后一次给出一个布尔向量 $v_1, \dots, v_n$。在接收到 $v_i$ 时，算法必须在接收到 $v_{i+1}$ 之前输出 $Mv_i$。该猜想是，任何能够为所有 $i = 1, \dots, n$ 正确输出 $Mv_i$ 的算法必须消耗至少 $n^{3-o(1)}$ 的时间。注意，如果所有向量 $v_1, \dots, v_n$ 是同时给出的，那么算法可以通过使用快速矩阵乘法在 $O(n^{\omega+o(1)})$ 时间内计算所有 $Mv_i$。OMv 问题也有一个自然的组合解释。为了翻译，令 $M$ 为二分图 $G = (V = L \cup R, E)$ 的邻接矩阵，算法接收并对其进行预处理。然后，按顺序给出 $n$ 个子集 $B_1, \dots, B_n \subseteq R$。在接收到 $B_i$ 后，算法必须返回 $L$ 中的哪些顶点在 $B_i$ 中至少有一个邻居。在假设 OMv 猜想的前提下，已经证明了几个动态问题的下限，包括即使在部分动态设置下的精确动态匹配。

在本文中，我们研究了 OMv 类现象与 $(1 - \epsilon)$-近似全动态匹配问题之间的联系，据我们所知，此前尚未对此进行过显式探讨。我们还研究了对偶的 $(1 + \epsilon)$-近似顶点覆盖问题。为此，我们表明，当且仅当对我们引入的动态近似 OMv 存在高效算法时（见定义 1.4 和定理 2），针对自适应对手的动态匹配才能实现理想的摊销运行时间 $n^{1-c}\epsilon^{-C}$。
