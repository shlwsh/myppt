# 学术/科技文献翻译专家 (Academic PDF Translator)

## 概述 (Overview)
本技能是一套专为 Agent 设计的英文学术与科技 PDF 文献高质量中文翻译指南。它确立了一套标准的工作流和翻译规范，能让 Agent 在不借助外部脚本的情况下，利用系统内置工具（如 `view_file`）高效、准确地将复杂的英文文献翻译成符合中文学术出版规范的 Markdown 文件。

本技能的核心目标是：
1. **学术级翻译质量**：译文语境自然、流畅，术语严谨，符合中文相关专业（如机器人、科技、医学、工程）的学术阅读习惯。
2. **格式高保真**：严格维护公式、表格、结构化排版和引用的完整性。
3. **参考文献汉化**：自动翻译引用的论文标题，实现中英对照。

## 依赖 (Dependencies)
本技能为纯文本指导型技能，完全依靠 Agent 的推理和执行能力。执行过程中需要配合以下内置工具：
- **`view_file`**：用于载入和解析 PDF 文件的页码与 OCR 文本内容。
- **语言模型（LLM）多语种翻译能力**：用于学术翻译、LaTeX 校验和参考文献处理。

## 快速开始 (Quick Start)
当用户请求翻译某个 PDF 文件时，执行以下命令模式：
1. 获取输入的源文件路径（如 `[源文件目录]/[源文件名].pdf`）。
2. **确定默认输出路径与新文件名**：
   - 输出目录默认为源文件所在目录下的 `docs-zh` 子目录（即 `[源文件目录]/docs-zh/`），如果该子目录不存在，Agent 应自动创建它。
   - **新文件名命名规则**：
     - **首选命名**：默认使用翻译后新文档的**中文主标题（Main Title）**作为新文件的名称，扩展名改为 `.md`。例如：如果文档主标题翻译为“基于阻抗参数预测的上肢康复机器人自适应阻抗控制技术研究”，则文件名应为 `基于阻抗参数预测的上肢康复机器人自适应阻抗控制技术研究.md`。
     - **备用命名（回退机制）**：若无法从文档中提取、获得或确定中文主标题（例如文献无明确标题，或主标题中包含不合法的操作系统文件名特殊字符），则翻译后的 Markdown 文件名默认为“源文件名”加上 `_zh` 后缀，扩展名改为 `.md`，即 `[源文件名]_zh.md`。
   - *示例*：输入源文件为 `doctor/pdf/paper_05.pdf` 时，提取并翻译出的中文主标题为“基于阻抗参数预测的上肢康复机器人自适应阻抗控制技术研究”，则默认输出路径为 `doctor/pdf/docs-zh/基于阻抗参数预测的上肢康复机器人自适应阻抗控制技术研究.md`。
   - *注*：除非用户在提示词中显式指定了其他输出路径或文件名，否则一律使用上述默认规则。
3. 调用 `view_file` 工具读取该 PDF，查看每一页的 OCR 文本。
4. 按照“步骤工作流（Workflow）”对每页内容进行逐页/逐段翻译和润色。
5. 将翻译结果以 Markdown 格式写入上述默认目标文件。

## 工作流步骤 (Workflow)

### 1. 文献载入与页码流式读取
- **执行方式**：使用 `view_file` 工具读取指定的英文 PDF 绝对路径。
- **处理要求**：
  - 若 PDF 页面较长或包含多页，必须使用 `view_file` 逐页或分批次读取。
  - 严禁遗漏任何页面的内容，确保翻译的连贯性与完整性。

### 2. 专业术语的双语对照标注
- **规则要求**：
  - 提取文献中的所有核心学术和科技名词。
  - 在每个术语**第一次出现时**，翻译为对应的规范中文术语，并在括号内加注英文原文。例如：`康复机器人（Rehabilitation Robot）`、`递推最小二乘法（Recursive Least Squares, RLS）`。
  - **后续再次出现时**，直接使用中文翻译，无需重复加注英文。

### 3. LaTeX 公式与图表的高精度还原
- **公式还原**：
  - 必须完整保留原文中的所有数学和物理公式，并统一排版为标准 LaTeX 格式（行内公式用单 `$` 符号包裹，块级公式用双 `$$` 符号包裹）。
  - **OCR 容错校正**：OCR 对数学符号极易产生误读（例如将 $\ddot{X}$ 识别为 `X€` 或 `X_`，将 $\pm$ 误读为 `±`，将减号 `-` 误读为加号 `+`）。Agent 必须根据该领域的物理/数学先验知识和上下文逻辑，手动推导和还原出 100% 正确的 LaTeX 公式。
- **表格与图表还原**：
  - 所有的表格数据必须以 Markdown 表格语法进行重建，列对齐清晰，包含中英文列名翻译。
  - 所有的插图（Figure）说明需以 `**图 X** [说明文字的中文翻译]` 形式进行独立排版。

### 4. 参考文献（References）的同步汉化
- **处理规范**：
  - 保留文末参考文献列表的原始引用格式和排版结构。
  - 对参考文献中的**论文标题（Paper Title）**进行中文翻译，并用中括号 `[...]` 的形式追加在该英文标题的后方。
  - **非标题部分**（例如作者姓名、期刊缩写、卷号、页码、年份及 DOI 地址）**必须保持英文原样**，不得进行翻译，以维护学术文献检索的可用性。
  - *示例*：
    - 原始：`Albu-Schaffer, A. (2007). A unified passivity based control framework for flexible joint robots. Springer Trac. Adv. Ro 28, 5-21.`
    - 译后：`Albu-Schaffer, A. (2007). A unified passivity based control framework for flexible joint robots [一种用于柔性关节机器人统一的无源性控制框架]. Springer Trac. Adv. Ro 28, 5-21.`

### 5. 高纯度学术译文输出
- **排版要求**：
  - 保持与原文 1-to-1 的层级标题（`#`、`##`、`###`）和段落结构。
  - 写入最终的 Markdown 目标文件时，**只输出翻译后的中文学术内容**。
  - 严禁在文件中加入任何非学术的“碎碎念”、Agent 状态输出或过渡性问候（如“这是为您翻译的第1页”）。

## 避坑指南 (Common Mistakes)

1. **OCR 残损公式直译**：
   * *错误*：直接输出 RLS 计算公式为 `K(k) = P(k-1)φ(k) / \lambda + \varphiT`。
   * *正确*：使用正确的 LaTeX 语法校正为 `$$K(k) = \frac{P(k-1)\varphi(k)}{\lambda + \varphi^{T}(k)P(k-1)\varphi(k)}$$`。

2. **参考文献过度翻译**：
   * *错误*：将作者 `Guo, Y. D.` 翻译为 `郭 Y. D.`，或将期刊名 `Neural Comput. Appl.` 翻译为 `神经网络计算与应用`。
   * *正确*：仅翻译具体文章名，作者与期刊/图书出版信息一律保持英文原版。

3. **对话文本泄露**：
   * *错误*：在最终写入目标 Markdown 文件时，文件的头部或尾部留有“好的，下面是为您翻译的内容：”或“希望本次翻译对您有帮助！”等对话用语。
   * *正确*：生成的 `.md` 文件必须是纯粹、严谨的学术出版级译本。
